Overview

In this assignment, I implemented several fundamental geometry processing algorithms, including Bezier curve evaluation, Bezier surface evaluation, area-weighted vertex normals, edge flip, edge split, and Loop subdivision. Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

In this part, I implemented de Casteljau’s algorithm to evaluate Bezier curves using recursive linear interpolation. Given a set of control points and a parameter t between 0 and 1, the algorithm repeatedly interpolates between adjacent control points until only a single point remains. This final point lies on the Bezier curve. My implementation constructs intermediate levels explicitly. At each recursion level, I compute a new set of interpolated points. This makes it easier to visualize the algorithm step by step. The scroll wheel controls the value of t. Changing t moves the evaluated point continuously along the curve. Dragging control points updates the entire curve dynamically, which verifies that the implementation responds correctly to changes in geometry. Run: ./meshedit ../bzc/curve1.bzc Press E repeatedly to show subdivision levels.

Part 2: Bezier surfaces with separable 1D de Casteljau

To evaluate Bezier surfaces, I extended the 1D de Casteljau algorithm into two dimensions using a separable approach. First, I evaluated Bezier curves along one parametric direction to obtain intermediate control points. Then I applied de Casteljau again along the other direction. This approach reduces 2D surface evaluation into repeated 1D curve evaluations, which simplifies implementation and ensures correctness. Run: ./meshedit ../bez/teapot.bez

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

I implemented Vertex::normal() by iterating over the one-ring of a vertex and summing area-weighted face normals using cross products. After accumulating all weighted normals, I normalized the result to obtain the final unit normal vector for shading. The difference between flat shading and area-weighted shading can be clearly seen below. Run: ./meshedit ../dae/teapot.dae

Part 4: Edge flip

I implemented edge flipping by modifying the connectivity of the two triangles that share the selected edge. The operation replaces the diagonal of a quadrilateral formed by the two triangles with the other diagonal. During implementation, I carefully reassigned all half-edge pointers, including next, twin, vertex, edge, and face references. Maintaining consistent connectivity was crucial to avoid breaking manifold structure. Run: ./meshedit ../dae/teapot.dae

Part 5: Edge split

For edge splitting, I inserted a new vertex at the midpoint of the selected edge. The two original adjacent triangles were replaced with four new triangles. This required creating new half-edges, edges, and faces while carefully updating connectivity relationships. I stored references to all relevant half-edges before modifying them to avoid losing track of connectivity during updates. Run: ./meshedit ../dae/teapot.dae

Part 6: Loop subdivision for mesh upsampling

I implemented Loop subdivision following the recommended multi-step procedure: (1) compute new positions for all original vertices using the Loop weighting rule based on their valence, (2) split every original edge, (3) flip new edges connecting old and new vertices, (4) update vertex positions. Run: ./meshedit ../dae/cube.dae

Extra Credit: Modified Butterfly Subdivision

I implemented the Modified Butterfly subdivision scheme as an alternative to Loop subdivision. Unlike Loop subdivision, Modified Butterfly is an interpolatory scheme, meaning original vertices remain fixed after subdivision.
